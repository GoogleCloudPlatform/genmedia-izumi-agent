# Architectural Walkthrough: Creative Studio Integration & Ads-X Enhancements

This document provides a business logic and architectural walkthrough of all enhancements implemented across `mediagent_kit` and the `Ads-X` AI Creative Director agent.

---

## 1. Storyboard Service & Persistence Layer (`CSStoryboardService`)

### Business Rationale

The Storyboard represents the core creative architecture of an ad campaign—defining scene sequences, narrative arcs, visual descriptions, voiceover scripts, background music, and reference assets.

### Key Logic & Capabilities

- **Explicit Functional Returns**: Storyboard modification helpers use explicit instance methods with returned data structures rather than mutating state objects in-place.
- **Media Asset Categorization**: Explicitly segregates AI-generated assets (`media_item_id` / `media_item`) from user-uploaded assets (`source_asset_id` / `source_asset`).
- **Backend Auto-Synchronization**: Automatically synchronizes modified storyboards with the Creative Studio backend (`POST /api/storyboards`), ensuring the web UI always reflects the agent's latest creative decisions.

---

## 2. Timeline Service & Cloud Video Rendering (`CSTimelineService`)

### Business Rationale

Timelines represent non-linear editing (NLE) structures holding video clips, audio placements, trims, volume/speed settings, and transition effects.

### Key Logic & Capabilities

- **Unified Workbench Contract**: Implements `CSTimelineService` targeting Creative Studio REST endpoints (`/api/workbench/timelines`).
- **Atomic Creation**: Supports creating timelines populated with complete video clips, audio placements, and transition specifications in a single request.
- **Cloud-Based Rendering**: Delegates video stitching to Creative Studio's backend rendering engine (`POST /api/workbench/timelines/{id}/render`) using cloud FFmpeg workers, avoiding heavy local rendering on the agent host.
- **Bi-directional Mapping**: Automatically converts between SDK asset models (`generated` / `uploaded`) and Creative Studio asset types (`media_item` / `source_asset`).

---

## 3. Configuration & Service Factory Architecture

### Business Rationale

Maintains a single source of truth for runtime settings, credentials, and GCP infrastructure parameters.

### Key Logic & Capabilities

- **Global Singleton Access**: Exported `mediagent_kit.services.aio.get_config()` to allow all tools and services to access the process-wide configuration initialized at server startup.
- **Drift Prevention**: Eliminates ad-hoc instantiation of default configuration objects, ensuring tools read active environment flags (such as `use_creative_studio`).

---

## 4. Dual-Mode Deep Linking & Creative Studio UI Integration

### Business Rationale

Provides seamless user navigation from agent chat outputs directly into the Creative Studio web application.

### Key Logic & Capabilities

- **Creative Studio Navigation**: When Creative Studio is enabled, video stitching tools generate Markdown links directly to:
  - 🎬 **Workbench NLE Timeline Editor**: `/workbench?timeline_id={id}`
  - 🖼️ **Rendered Video Asset Gallery**: `/gallery/{id}`
- **Fallback Safety**: Maintains legacy HTML Canvas deep links when operating in standalone Izumi Native mode.

---

## 5. Multimodal Reference Image Conditioning (`scene_generation_utils.py`)

### Business Rationale

Ensures generated scene first frames faithfully preserve visual details from user-uploaded product images (e.g., `product.png`) or AI-casted characters (e.g., `virtual_creator_123.png`).

### Key Logic & Capabilities

- **AssetRef Resolution**: Automatically inspects reference asset filenames/IDs attached to scene prompts and resolves them against the workspace asset repository.
- **Multi-Modal Model Inputs**: Passes resolved `reference_assets` into `mediagen_service.generate_image(...)`. Creative Studio's image generator maps these into `sourceAssetIds` and `sourceMediaItems` to condition Imagen 3 / Gemini image generation.

---

## 6. Multi-Scenario Ingestion & ADK Blob Interceptor (`blob_interceptor_callback`)

### Business Rationale

Captures and catalogs any user-provided media—whether uploaded via chat attachments, frontend file pickers, or explicit asset tags—making them immediately available as visual references for ad generation.

### Key Logic & Capabilities

- **Unified Ingestion API**: Uses `asset_service.upload_asset(...)` across all ingestion pathways.
- **Scenario A (Inline Multimodal Attachments)**: Intercepts raw chat attachments, uploads the binary bytes to Creative Studio, generates visual descriptions, and catalogs them in campaign state.
- **Scenario B (UI Session Artifacts)**: Intercepts `<start_of_user_uploaded_file: file.png>` tags generated by frontend upload controls, loads raw bytes from context, uploads to Creative Studio, and catalogs visual descriptions.
- **Scenario C (Creative Studio Asset Tag References)**: Intercepts `<creative_studio_asset id="119" type="source_asset" />` tags, fetches asset metadata from Creative Studio, and registers asset details into campaign state.
- **Unified Visual Description**: Uses a standardized visual summary prompt (`DESCRIPTION_INSTRUCTION`) across all interceptors to generate concise product summaries for downstream Storyboard reasoning.
